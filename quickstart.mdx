---
title: "Quickstart: install and configure i915 SR-IOV DKMS"
description: "Install i915 SR-IOV DKMS, configure kernel parameters for i915 or xe, create Virtual Functions, and verify GPU passthrough is working on Linux."
---

This guide walks you through the fastest path to a working SR-IOV setup on a supported Linux host. You will install the `i915-sriov-dkms` module, configure kernel parameters to enable SR-IOV on the physical GPU, create Virtual Functions, and verify that the VFs appear on the PCI bus and in the kernel log.

<Steps>
  <Step title="Install prerequisites">
    Install the kernel headers for your running kernel along with the DKMS framework and build tools.

    <Tabs>
      <Tab title="Debian / Ubuntu">
        ```bash
        sudo apt update
        sudo apt install build-essential dkms git linux-headers-$(uname -r)
        ```
      </Tab>
      <Tab title="Arch Linux">
        ```bash
        sudo pacman -S base-devel dkms git linux-headers
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Install the DKMS module">
    <Tabs>
      <Tab title="Debian / Ubuntu (.deb)">
        Download and install the pre-built `.deb` package from the [GitHub releases page](https://github.com/strongtz/i915-sriov-dkms/releases). Replace the filename with the current release version:

        ```bash
        sudo dpkg -i i915-sriov-dkms_2026.03.05_amd64.deb
        ```

        DKMS will automatically build the module for your running kernel after installation.
      </Tab>
      <Tab title="Manual (git)">
        Clone the repository and add it as a DKMS source, then build and install:

        ```bash
        git clone https://github.com/strongtz/i915-sriov-dkms.git
        sudo dkms add ./i915-sriov-dkms
        sudo dkms install i915-sriov-dkms/2026.03.05
        ```
      </Tab>
      <Tab title="Arch Linux (AUR)">
        Install from the AUR using your preferred AUR helper:

        ```bash
        yay -S i915-sriov-dkms
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Set kernel parameters">
    Add the appropriate parameters to your bootloader. Choose whether you want to use the `i915` driver or the `xe` driver.

    <Warning>
      The `xe` driver does **not** support Meteor Lake (MTL) or Lunar Lake (LNL) GPUs. Use the `i915` parameters if you have one of these platforms.
    </Warning>

    <Tabs>
      <Tab title="i915 driver">
        ```
        intel_iommu=on i915.enable_guc=3 i915.max_vfs=7 module_blacklist=xe
        ```

        - `intel_iommu=on` — enables the IOMMU required for VF isolation
        - `i915.enable_guc=3` — enables GuC submission, required for SR-IOV on i915
        - `i915.max_vfs=7` — allocates resources for up to 7 Virtual Functions
        - `module_blacklist=xe` — prevents the xe driver from loading alongside i915
      </Tab>
      <Tab title="xe driver">
        First, find your GPU's device ID:

        ```bash
        cat /sys/devices/pci0000:00/0000:00:02.0/device
        ```

        Then set the kernel parameters, replacing `${device_id}` with the output of the command above:

        ```
        intel_iommu=on xe.max_vfs=7 xe.force_probe=${device_id} module_blacklist=i915
        ```

        - `intel_iommu=on` — enables the IOMMU required for VF isolation
        - `xe.max_vfs=7` — allocates resources for up to 7 Virtual Functions
        - `xe.force_probe=${device_id}` — force-enables probe for your specific GPU
        - `module_blacklist=i915` — prevents the i915 driver from loading alongside xe
      </Tab>
    </Tabs>
  </Step>

  <Step title="Update the bootloader and initramfs">
    Apply the kernel parameters you added and regenerate your initramfs so the changes take effect on next boot.

    <Tabs>
      <Tab title="GRUB (Debian / Ubuntu)">
        Open `/etc/default/grub` and append the parameters to `GRUB_CMDLINE_LINUX_DEFAULT`, then update GRUB and regenerate initramfs:

        ```bash
        sudo update-grub
        sudo update-initramfs -u
        ```
      </Tab>
      <Tab title="GRUB (Arch Linux)">
        Open `/etc/default/grub`, append the parameters to `GRUB_CMDLINE_LINUX_DEFAULT`, then:

        ```bash
        sudo grub-mkconfig -o /boot/grub/grub.cfg
        sudo mkinitcpio -P
        ```
      </Tab>
      <Tab title="systemd-boot">
        Open the loader entry file for your kernel (typically under `/boot/loader/entries/`) and append the parameters to the `options` line:

        ```conf
        options root=... rw intel_iommu=on i915.enable_guc=3 i915.max_vfs=7 module_blacklist=xe
        ```

        No additional command is needed — systemd-boot reads the file on each boot.
      </Tab>
    </Tabs>
  </Step>

  <Step title="Enable Virtual Functions at boot">
    Configure the system to create VFs automatically each time it boots. The sysfs path below assumes your Intel GPU is at PCI address `0000:00:02.0`, which is standard on most systems.

    <Tabs>
      <Tab title="sysfsutils (Debian / Ubuntu)">
        Install sysfsutils and add a rule to create 7 VFs at boot:

        ```bash
        sudo apt install sysfsutils
        echo "devices/pci0000:00/0000:00:02.0/sriov_numvfs = 7" | sudo tee -a /etc/sysfs.conf
        ```
      </Tab>
      <Tab title="tmpfiles.d (Arch Linux / systemd)">
        Create a tmpfiles rule that writes to the sysfs VF count node at boot:

        ```bash
        echo 'w /sys/devices/pci0000:00/0000:00:02.0/sriov_numvfs - - - - 7' | \
          sudo tee /etc/tmpfiles.d/i915-sriov-numvfs.conf
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Reboot and verify">
    Reboot your system, then confirm that the VFs have been created and the module loaded correctly.

    Check the kernel log for i915 SR-IOV messages:

    ```bash
    sudo dmesg | grep i915
    ```

    List PCI devices to confirm the VFs appear:

    ```bash
    lspci -nnk | grep -A2 "VGA\|Display"
    ```

    You should see the Physical Function (PF) at `00:02.0` and up to 7 Virtual Functions listed below it (e.g., `00:02.1` through `00:02.7`).

    <Warning>
      The Physical Function (`00:02.0`) must **never** be passed through to a VM. Only pass individual Virtual Functions to guests. Passing the PF to a VM will break host GPU access and can destabilize the system.
    </Warning>
  </Step>
</Steps>

<Note>
  You must also install `i915-sriov-dkms` on every guest VM that will use a Virtual Function. The guest kernel needs the patched i915 or xe module to communicate with the VF. See the [Linux guest installation](/install/linux-guest) guide for details.
</Note>
