i915 SR-IOV DKMS is a community-maintained DKMS kernel module that patches the Intel i915 and xe GPU drivers with SR-IOV (Single Root I/O Virtualization) support. It enables you to create up to 7 Virtual Functions (VFs) from a single Intel iGPU, allowing multiple virtual machines to share hardware-accelerated graphics — useful for Proxmox, KVM, and other hypervisor setups.

<Warning>
  This module is **highly experimental** and is a community project not affiliated with Intel. Install only if you understand the risks. You must install the module on **both the host and every guest VM**.
</Warning>

## What is SR-IOV?

SR-IOV (Single Root I/O Virtualization) is a PCI-SIG standard that allows a single physical PCIe device — in this case an Intel integrated GPU — to present itself as multiple virtual devices. Each Virtual Function (VF) can be assigned to a separate VM, giving that VM direct hardware access for accelerated graphics, video transcoding, and compute workloads.

This module ports SR-IOV support from Intel's internal `mainline-tracking` kernel tree back to standard kernels via DKMS, so it rebuilds automatically on kernel upgrades.
